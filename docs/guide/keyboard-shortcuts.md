# Keyboard shortcuts

`Cmd` on macOS, `Ctrl` on Windows/Linux.

## App-wide

| Shortcut | Action |
|---|---|
| `Cmd/Ctrl+N` | New connection |
| `Cmd/Ctrl+Shift+C` | List connections |
| `Cmd/Ctrl+,` | Settings |
| `Cmd/Ctrl+R` | Soft reload (refresh sidebar tree + active tab's data) |
| `Cmd/Ctrl+Shift+R` | Hard reload (full page reload) - works on every screen |
| `Cmd/Ctrl+W` | Close the active tab |

## SQL editor

| Shortcut | Action |
|---|---|
| `Cmd/Ctrl+Enter` | Run the statement under the cursor |
| `Cmd/Ctrl+Shift+Enter` | Run all statements |
| `Cmd/Ctrl+I` | Load a query file |
| `Cmd/Ctrl+S` | Save the query buffer |
| `Cmd/Ctrl+Shift+S` | Save query as… |

## Data (File menu)

| Shortcut | Action |
|---|---|
| `Cmd/Ctrl+Shift+I` | Import data… |
| `Cmd/Ctrl+Shift+E` | Export data… |

## Data grid

| Shortcut | Action |
|---|---|
| `Cmd/Ctrl+Z` | Undo staged edit |
| `Cmd/Ctrl+Shift+Z` / `Cmd/Ctrl+Y` | Redo |
| `Cmd/Ctrl+S` | Commit staged edits (when a data tab is active) |

> Some shortcuts are context-sensitive: the editor's `Cmd/Ctrl+S` saves the
> query buffer, while the data grid's `Cmd/Ctrl+S` commits row edits. The active
> tab decides which applies.
